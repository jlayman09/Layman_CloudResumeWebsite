The Project: AWS Cloud Resume Challenge

AWS Serverless Portfolio: Engineering a full-stack CI/CD pipeline with IaC.

January 28, 2026

The Cloud Resume Challenge is much more than a digital resume; it is a rigorous, sixteen-step project specification designed to bridge the gap between "learning" and "doing". Instead of following a step-by-step tutorial, I am building a full-stack, serverless application based on a professional project spec that requires me to solve complex architectural problems independently.

My Project Objectives

  • Bridge My Skillset: Leveraging my experience as an Intel Analyst and educator to master the "engineered prints" of cloud architecture.
  • Master Serverless Architecture: Building a cost-effective, scalable API using AWS Lambda, API Gateway, and DynamoDB.
  • Infrastructure as Code (IaC): Moving away from manual configuration to define every resource in code, ensuring my environment is reproducible and professional.
  • CI/CD Automation: Implementing a "hands-off" deployment pipeline where every code change is automatically tested and pushed to production via GitHub Actions.
  • Security-First Mindset: Implementing HTTPS, DNSSEC, and the principle of least privilege to ensure my cloud presence is battle-hardened.

Why I'm Taking This On?

In the cloud, certifications are just the baseline; real-world ability is proven by the ability to "Google well" and glue services together into a functioning system. This project serves as my "baptism by fire," allowing me to bang my head against DNS, CORS, and deployment scripts until the knowledge is burned into my brain.


Upcoming Projects

Phase 2: Launching My Site with AWS

The next phase of my website focuses on production-ready deployment and security. I'm implementing HTTPS across the entire site by placing my static S3 website behind Amazon CloudFront, ensuring encrypted traffic and modern web security standards. Once CloudFront is in place, I'll configure a custom domain and DNS routing using Amazon Route 53, pointing my personal domain directly to the CloudFront distribution. This step completes the transition from a basic static site to a secure, globally distributed web presence—mirroring real-world cloud deployment practices. This milestone marks the official “go-live” moment for my site.

Phase 3: Securing the Site with AWS CloudFront

This phase focuses on hardening the website by placing it behind Amazon CloudFront. I configured a global CDN in front of the S3 static site to enforce HTTPS, improve performance, and add an additional security layer. This involved setting up SSL certificates, configuring origin access controls, updating DNS records, and handling redirects to ensure all traffic is encrypted and routed correctly. The result is a faster, more secure, and production-ready website that follows real-world cloud security best practices.
